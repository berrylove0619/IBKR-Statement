# 宏观风险映射框架

**何时读取：** 仅在周度宏观复盘，或增长、通胀、流动性、政策、利率出现异常冲击并可能影响实际持仓时读取。

**何时不要读取：** 正常交易日、宏观数据没有异常变化，或事件无法映射到实际持仓时不要读取。

## 从冲击映射到持仓

只分析可验证的新变化。对每项冲击依次写出：**事实变化 → 传导渠道 → 持仓暴露 → 影响方向与期限 → 抵消因素**。遵守固定新闻证据 reference 已建立的证据门槛，不用宏观观点替代事件证据。

1. **增长**：判断需求、销量、订单、广告、企业支出或违约率如何传导到持仓收入和经营杠杆。
2. **通胀**：判断投入成本、工资、运费和定价权如何传导到毛利率、需求弹性和实际购买力。
3. **流动性**：判断信贷可得性、融资条件和风险偏好如何传导到估值倍数、再融资和高久期资产。
4. **政策**：判断财政、监管、贸易、税收或产业政策是否改变持仓的成本、市场准入、补贴或终端需求。
5. **利率**：判断折现率、债务成本、存款/信贷收益、汇率和住房或资本开支需求如何传导到持仓。

## 组合输出

- 只展开存在直接暴露或清晰宏观共同因子的持仓；其余记为“无实质关联”。
- 合并受同一因子驱动的持仓，评估净暴露、集中度、方向相反的对冲和影响期限，避免逐股重复同一宏观叙事。
- 区分已发生冲击、市场已定价预期和仍待验证的情景；不要把预测写成事实。
- 用一张简表输出“因子、证据、传导、受影响持仓、方向、期限、反证”，不要写泛宏观长文。
- 只有达到证据门槛且传导到实际持仓的变化才可支持风险动作；否则写“观察”或“维持原计划”。
