# 科技与供应链传导框架

**何时读取：** 仅当 AI 算力、半导体、数据中心、光通信或关键供应链事件命中实际持仓时读取。

**何时不要读取：** 只有泛行业热度、主题行情、社交媒体叙事，或无法建立持仓暴露证据链时不要读取。

## 判定关联层级

1. **一阶**：事件直接改变持仓公司的订单、售价、出货、成本、产能、资本开支、监管约束或已披露客户关系。
2. **二阶**：事件先影响可识别的客户、供应商或关键瓶颈，再通过明确商业关系和收入暴露传导到持仓。
3. **无关联**：只有行业标签相同、竞争者新闻、概念映射或传导链缺环；保留扫描记录，不进入持仓建议。

## 建立证据链

对每个命中依次验证并记录：

- **客户**：谁采购、采购什么、需求变化是否已确认，持仓公司是否真正受益或受损。
- **供应商**：谁提供关键部件、设备、材料或服务，替代性、库存和交付约束是什么。
- **产能**：新增、受限或转移的产能位于哪一环，投产时间、利用率、良率和资本强度如何。
- **收入暴露**：相关产品、客户或地区占收入、订单或利润的比例；无法量化时明确标记未知，不用总市场规模代替公司暴露。

用“事件 → 客户/供应商 → 产能或订单变化 → 持仓收入/利润暴露 → 时间范围”的完整链条支持二阶判断。链条任一关键环节缺证据时降为“无关联”或“观察”。

## 分主题检查

- **AI 算力**：区分训练、推理、加速器、服务器和电力需求，避免把所有 AI 支出视为同一受益面。
- **半导体**：定位设计、晶圆制造、封装测试、设备、材料和存储环节，检查制程、良率、库存与周期位置。
- **数据中心**：检查云厂商资本开支、机架交付、电力、冷却、网络和建设周期，不把公告容量等同已实现收入。
- **光通信**：检查速率升级、光模块/器件、交换网络、客户认证、供货份额与价格压力。
- **关键供应链**：检查单一来源、地域集中、出口限制、交期、替代认证和库存缓冲。

## 动作纪律

遵守固定新闻证据 reference 已建立的证据门槛，并优先用公司披露、客户/供应商披露和监管文件闭合关系。泛行业热度、市场规模预测、单一订单传闻或股价联动不能触发交易。

只有证据链完整的一阶或二阶影响才可进入条件式动作，并写明触发条件、反证条件、时间范围和主要风险；无完整链条时写“观察”或“维持原计划”。
