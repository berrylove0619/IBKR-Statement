# 持仓分析规则

## 持仓快照门槛

只使用 IBKR-Statement 最新一次状态为成功的导入，不使用最新一次失败尝试，也不从新闻、记忆或历史回答猜测持仓。

记录并展示：成功导入时间、持仓截至时间、源文件标识、账户币种、持仓数量和数据年龄。若快照未覆盖最近一个已结束的美股交易日，将其标为“过期”；若无法判断交易日或时间，标为“新鲜度未知”。缺失、过期或新鲜度未知时，仍可报告市场事实，但不得给出个股加减仓动作，并明确要求刷新导入。

缺少最新成功导入时，题设、对话或用户消息中列出的证券只组成“用户提供的临时范围”。可对该范围做有限事件映射，但不得把它当作 IBKR 实际持仓，不得据此填写正式 `scanned_holdings / total_holdings`、组合权重或“全持仓覆盖”；正式覆盖统一写“未验证”。

## 全持仓扫描

1. 枚举快照中的全部非零持仓，保留代码、名称、资产类型、数量、市值、成本和组合权重；字段缺失时显式标记。
2. 对每个持仓匹配已验证事件，完成四类关联标注。
3. 仅在最新成功导入存在时记录正式 `scanned_holdings / total_holdings`。无法分析的正式持仓计入遗漏清单并写明原因，不得当作“无变化”。没有成功导入时，另列临时范围项目数，不生成正式覆盖分数。
4. 按证据强度、影响路径、组合权重、时效和潜在损失排序，最多展开 6 个重点持仓；其余合并为“无重大变化”。

## 四类关联

每个“事件 × 持仓”只给一个主要类别：

| 类别 | 判断标准 | 报告去向 |
|---|---|---|
| 直接 | 发行人公告、监管文件、财报、指引、资本行动或该证券的合同条款直接改变持仓事实 | 可进入持仓建议 |
| 二阶 | 事件通过已命名客户、供应商、竞争者或关键投入传导，并能写出具体、可验证的因果链 | 路径清晰时可进入持仓建议 |
| 宏观共同因子 | 利率、通胀、汇率、能源、流动性或政策同时影响多个资产，但没有公司特定新事实 | 进入组合风险，不单独触发个股动作 |
| 无实质关联 | 只有主题、关键词或行业标签重合，无法给出可验证传导路径 | 只计覆盖，不进入建议 |

二阶路径必须写成“事件 → 中间变量 → 持仓收入、成本、资本开支、估值或风险”的链条，并为关键连接提供证据。不能写出链条时归为无实质关联。

## 组合级综合

完成全扫描后只综合一次：

- 计算或披露重点持仓权重、前五大集中度、行业／主题聚集和共同宏观暴露；数据不足时不得伪造精确数值。
- 区分“事件改变投资逻辑”与“价格短期波动”。价格变化本身不是仓位动作证据。
- 将新闻触发动作与既有集中度再平衡分开。后者标为“组合风险动作”，说明它不由当日新闻单独触发。
- 只有持仓公司财报命中且常规综合无法回答关键差异时，允许增加一次财报专项分析。

## 条件式动作

动作只使用“观察 / 持有 / 条件式加仓 / 条件式减仓 / 风险控制 / 维持原计划”。每项动作必须包含：

- `trigger`：未来可观察、可验证的触发条件；
- `disconfirm`：使判断失效或反转的反证条件；
- `horizon`：盘前／数日／至下次财报／长期等时间范围；
- `risk`：证据、业务、估值、流动性、汇率、税务或执行风险中的主要项；
- `evidence_status`：动作依据对应的新闻验证状态；
- `impact_class`：直接或二阶；
- `rationale`：动作与实际持仓权重及传导路径的关系。

仅 `verified_primary_plus_independent` 或 `verified_two_independent` 且属于直接／清晰二阶的事件可支持条件式仓位动作。单一来源、匿名爆料、传闻、短时价格、泛行业新闻和无实质关联不得独立触发加仓、减仓或清仓。没有新证据或触发条件未满足时，输出“维持原计划”。

没有任何候选 URL 或来源时，记录 `evidence_gap`，并把验证状态写为“不适用”。`evidence_gap` 只是证据缺口标记，不属于四种验证状态，不能支持任何动作依据。零来源不得标为 `unverified_single_source`。

即使快照或字段缺失，每个被选为重点持仓的条目仍保留 `trigger`、`disconfirm`、`horizon`、`risk`、`evidence_status`、`impact_class` 和 `rationale`；按实际情况填写“未提供”“未验证”或“不适用”，不得省略字段。

## 风险披露

至少检查持仓数据过期、证据冲突、集中度、相关性聚集、财报跳空、流动性、汇率和税务影响。输出是研究辅助，不是收益保证；不得自动下单，任何动作由用户确认后在 IBKR 自行执行。
