# 持仓分析规则

## 插件快照门槛

只使用 IBKR 输入合同中三项固定插件调用形成的当次快照，不使用本地文件、新闻、记忆或历史回答猜测持仓。

- `ready`：正式扫描全部非零持仓，并可使用净清算值、现金、保证金和币种余额做完整组合分析。
- `partial`：正式扫描插件返回的全部非零持仓，但缺失字段写“未验证”；只允许“观察／持有／刷新插件数据后复核”，禁止条件式加仓和禁止条件式减仓。
- `empty`：确认没有未平仓持仓，不生成持仓建议。
- `source_unavailable`：正式覆盖写“未验证”，停止持仓建议，且不得回退其他来源。

记录并展示插件查询时间、账户数据状态、新鲜度、持仓数量和缺失组件。`stale` 或 `unknown` 时仍可报告市场事实，但持仓动作统一限制为“观察／刷新插件数据后复核”。

用户消息中列出的证券只组成“用户提供的临时范围”。它不得写入正式 `scanned_holdings / total_holdings`、组合权重或“全持仓覆盖”。

## 全持仓扫描

1. 在收集新闻前枚举插件返回的全部非零持仓，保留代码、合约标识、资产类型、数量、市值、平均成本、当日与未实现盈亏、币种和可验证权重。
2. 仅在市场价值与净清算值均有效且净清算值非零时计算 `portfolio_weight`；缺失价格、成本、币种或盈亏时保留空值。
3. 对每个正式持仓匹配新闻事件并完成四类关联标注；无法分析的持仓计入遗漏清单并写明原因。
4. 按证据强度、影响路径、组合权重、时效和潜在损失排序，最多展开 6 个重点持仓；其余合并为“无重大变化”。
5. 只有新闻初筛命中重点持仓且现有返回不足时，才条件查询额外行情；不得为全部持仓重复调用行情。

## 四类关联

每个“事件 × 持仓”只给一个主要类别：

| 类别 | 判断标准 | 报告去向 |
|---|---|---|
| 直接 | 发行人公告、监管文件、财报、指引、资本行动或该证券的合同条款直接改变持仓事实 | 可进入持仓建议 |
| 二阶 | 事件通过已命名客户、供应商、竞争者或关键投入传导，并能写出具体、可验证的因果链 | 路径清晰时可进入持仓建议 |
| 宏观共同因子 | 利率、通胀、汇率、能源、流动性或政策同时影响多个资产，但没有公司特定新事实 | 进入组合风险，不单独触发个股动作 |
| 无实质关联 | 只有主题、关键词或行业标签重合，无法给出可验证传导路径 | 只计覆盖，不进入建议 |

二阶路径必须写成“事件 → 中间变量 → 持仓收入、成本、资本开支、估值或风险”的链条，并为关键连接提供证据。不能写出链条时归为无实质关联。

## 组合级综合

完成全扫描后只综合一次：

- `ready` 时计算或披露重点持仓权重、前五大集中度、现金比例、保证金余量、杠杆、行业／主题聚集和共同宏观暴露。
- `partial` 时不得伪造权重、现金、集中度或保证金结论，不生成规模相关动作。
- 区分“事件改变投资逻辑”与“价格短期波动”；价格变化本身不是仓位动作证据。
- 将新闻触发动作与既有集中度再平衡分开。后者标为“组合风险动作，非当日新闻触发”。
- 只有持仓公司财报命中且常规综合无法回答关键差异时，允许增加一次财报专项分析。

## 条件式动作

动作只使用“观察 / 持有 / 条件式加仓 / 条件式减仓 / 风险控制 / 维持原计划”。每项动作必须包含：

- `trigger`：未来可观察、可验证的触发条件；
- `disconfirm`：使判断失效或反转的反证条件；
- `horizon`：盘前／数日／至下次财报／长期等时间范围；
- `risk`：证据、业务、估值、流动性、汇率、税务或执行风险中的主要项；
- `evidence_status`：动作依据对应的新闻验证状态；
- `impact_class`：直接或二阶；
- `rationale`：动作与实际持仓权重及传导路径的关系。

仅在账户状态 `ready`、新鲜度 `fresh`、事件达到 `verified_primary_plus_independent` 或 `verified_two_independent`，且属于直接／清晰二阶时，才可支持条件式仓位动作。没有新证据或触发条件未满足时输出“维持原计划”。

单一来源、匿名爆料、传闻、短时价格、泛行业新闻和无实质关联不得独立触发仓位动作。没有候选来源时记录 `evidence_gap`，验证状态写“不适用”。

即使字段缺失，每个重点持仓仍保留 `trigger`、`disconfirm`、`horizon`、`risk`、`evidence_status`、`impact_class` 和 `rationale`，按实际情况填写“未提供”“未验证”或“不适用”。

## 风险披露

至少检查插件数据状态与时间、证据冲突、集中度、相关性聚集、财报跳空、流动性、保证金、汇率和税务影响。输出是研究辅助，不是收益保证；不得自动下单，任何动作由用户确认后在 IBKR 自行执行。
