# 持仓分析规则

## 插件快照门槛

只使用 IBKR 输入合同中三项固定插件调用形成的当次快照，不使用本地文件、新闻、记忆或历史回答猜测持仓。

- `ready`：正式扫描全部非零持仓，并可使用净清算值、现金、保证金和币种余额做完整组合分析。
- `partial`：正式扫描插件返回的全部非零持仓，但缺失字段写“未验证”；只允许“观察／持有／刷新插件数据后复核”，禁止条件式加仓和禁止条件式减仓。
- `empty`：确认没有未平仓持仓，不生成持仓建议。
- `source_unavailable`：正式覆盖写“未验证”，停止持仓建议，且不得回退其他来源。

记录并展示插件查询时间、账户数据状态、新鲜度、持仓数量和缺失组件。`stale` 或 `unknown` 时仍可报告市场事实，但持仓动作统一限制为“观察／刷新插件数据后复核”。

用户消息中列出的证券只组成“用户提供的临时范围”。它不得写入正式 `scanned_holdings / total_holdings`、组合权重或“全持仓覆盖”。

## 全持仓扫描

1. 在收集新闻前枚举插件返回的全部非零持仓，保留代码、合约标识、资产类型、数量、原始市值及其币种口径、原币市值、基准币种、`fx_to_base`、基准币市值、平均成本、当日与未实现盈亏和可验证权重。
2. 仅在 `market_value_base` 与基准币种净清算值均有效且净清算值非零时计算 `portfolio_weight`；缺失价格、成本、币种口径、汇率或盈亏时保留空值，不跨币种直接相除。
3. 对每个正式持仓匹配新闻事件并完成四类关联标注；无法分析的持仓计入遗漏清单并写明原因。
4. 按证据强度、影响路径、组合权重、时效、潜在损失和轮换复核需求排序。正常日选择 4-5 个深度分析持仓，硬上限 5 个；只有重大事件日才可增加到 8 个。
5. 每个非零持仓都输出独立卡片。未进入深度分析的持仓使用简版卡片，明确写“今日无足以改变原计划的新证据”或对应的数据缺口，不得合并隐藏。
6. 只有新闻初筛命中深度分析持仓且现有返回不足时，才条件查询额外行情；不得为全部持仓重复调用行情。

## 深度分析触发

正式持仓非空时读取 `analysis-committee.md`，并为选中的深度持仓执行八角色分析。深度持仓至少满足一项：

- 命中已交叉验证的直接重大新闻；
- 命中已交叉验证且传导路径清晰的重大二阶事件；
- 距财报不超过 7 个自然日或刚发布财报；
- 单一持仓或共同因子集中风险显著；
- 公司投资逻辑出现可验证变化；
- 属于账户高权重持仓且已到每日轮换复核顺序；
- 用户在当次任务明确要求深挖。

深度持仓按“事件强度、组合权重、时间敏感性、上次深度复核时间”排序。优先覆盖直接重大事件、路径清晰的重大二阶事件与临近财报持仓，再从高权重和长期未深挖持仓中轮换补足；不得为了凑足 4 只而编造新事实。超过上限的持仓仍生成简版卡片，并说明未展开原因。

## 四类关联

每个“事件 × 持仓”只给一个主要类别：

| 类别 | 判断标准 | 报告去向 |
|---|---|---|
| 直接 | 发行人公告、监管文件、财报、指引、资本行动或该证券的合同条款直接改变持仓事实 | 可进入持仓建议 |
| 二阶 | 事件通过已命名客户、供应商、竞争者或关键投入传导，并能写出具体、可验证的因果链 | 路径清晰时可进入持仓建议 |
| 宏观共同因子 | 利率、通胀、汇率、能源、流动性或政策同时影响多个资产，但没有公司特定新事实 | 进入组合风险，不单独触发个股动作 |
| 无实质关联 | 只有主题、关键词或行业标签重合，无法给出可验证传导路径 | 只计覆盖，不进入建议 |

二阶路径必须写成“事件 → 中间变量 → 持仓收入、成本、资本开支、估值或风险”的链条，并为关键连接提供证据。不能写出链条时归为无实质关联。

## 组合级综合

完成全扫描后只综合一次：

- `ready` 时使用基准币市值计算或披露重点持仓权重、前五大绝对集中度、毛／净敞口、现金比例、保证金余量、杠杆、行业／主题聚集和共同宏观暴露；展示 `base_currency`、`fx_to_base` 与公式输入。默认公式：`portfolio_weight = market_value_base / net_liquidation_base`，`gross_exposure = sum(abs(market_value_base)) / net_liquidation_base`，`net_exposure = sum(market_value_base) / net_liquidation_base`，前三／前五集中度为按绝对基准币市值排序后的绝对权重之和。
- `partial` 时不得伪造权重、现金、集中度或保证金结论，不生成规模相关动作。
- 区分“事件改变投资逻辑”与“价格短期波动”；价格变化本身不是仓位动作证据。
- 将新闻触发动作与既有集中度再平衡分开。后者标为“组合风险动作，非当日新闻触发”。
- 只有持仓公司财报命中且常规综合无法回答关键差异时，允许增加一次财报专项分析。

## 持仓卡片与条件式动作

每张卡片先展示持仓事实：代码、名称、资产类别、币种、数量、市场价、原始市值及口径、原币市值、`base_currency`、`fx_to_base`、基准币市值、平均成本、账户权重、当日盈亏和未实现盈亏。插件没有返回的字段写“未提供”或“未验证”，不得推算。

持仓事实之后固定显示“持仓建议·大白话总结”。使用 `section.plain-language-summary` 包裹三个明确字段“当前动作”“买入建议”“卖出建议”，并提供恰好一个 `p.plain-summary-text`。正文为 100–200 个中文字符，用普通投资者可直接理解的语言概括当前结论、核心原因、主要仓位风险和下一项可验证条件；不堆砌术语、角色名称或证据编号。

大白话总结只是既有裁决的通俗表达，不建立新的动作门槛。`ready + fresh` 且事件证据达标时，买入或卖出建议可以表达条件式加仓或条件式减仓，并写清触发条件；`partial / stale / unknown` 时，买入建议只能是“暂不加仓／等待确认”，卖出建议只能是“暂不主动卖出／仅观察风险”，不得给出交易规模、精确价位或条件式动作。`source_unavailable / empty` 不生成个人买卖建议。

每张卡片随后展示：当日关联事件、影响类别、主要影响路径、组合风险、综合判断、触发条件、反证条件、时间范围和证据链接。深度卡片另外展示基本面变化、行业与竞争、市场预期差、最强看多理由、最强看空理由、对实际仓位的影响、投委会共识与分歧、未来观察指标；简版卡片不需要为了格式编造多角色结论。

深度卡片的四字段快速摘要保持展开；八角色详细分析放入不带 `open` 属性的 `details.committee-details`，标题固定为“展开八角色详细分析与证据”，默认折叠。简版卡片不需要创建空的八角色折叠区。

动作只使用“维持原计划 / 继续持有 / 谨慎持有 / 等待确认 / 暂缓加仓 / 条件式加仓 / 条件式减仓 / 优先控制组合风险”。每项动作必须包含：

- `trigger`：未来可观察、可验证的触发条件；
- `disconfirm`：使判断失效或反转的反证条件；
- `horizon`：盘前／数日／至下次财报／长期等时间范围；
- `risk`：证据、业务、估值、流动性、汇率、税务或执行风险中的主要项；
- `evidence_status`：动作依据对应的新闻验证状态；
- `impact_class`：直接或二阶；
- `rationale`：动作与实际持仓权重及传导路径的关系。

“已交叉验证”必须明确映射为 `verified_primary_plus_independent` 或 `verified_two_independent`，不得只使用自然语言标签代替正式枚举。仅在账户状态 `ready`、新鲜度 `fresh`、事件达到上述两种状态之一，且属于直接／清晰二阶时，才可支持新闻或公司事件驱动的条件式仓位动作。没有新证据或触发条件未满足时输出“维持原计划”。

组合风险动作不要求公司新闻达到双源门槛，但必须同时满足：账户状态 `ready`、新鲜度 `fresh`、权重／现金／保证金／杠杆等风险字段完整、计算口径可复核，并明确标注“组合风险动作，非当日新闻触发”。它只能使用“优先控制组合风险”或带组合风险条件的“谨慎持有”，不得借集中度结论声称公司基本面已经改变。

单一来源、匿名爆料、传闻、短时价格、泛行业新闻和无实质关联不得独立触发仓位动作。没有候选来源时记录 `evidence_gap`，验证状态写“不适用”。

即使字段缺失，每个持仓仍保留 `trigger`、`disconfirm`、`horizon`、`risk`、`evidence_status`、`impact_class` 和 `rationale`，按实际情况填写“未提供”“未验证”或“不适用”。

## 风险披露

至少检查插件数据状态与时间、证据冲突、集中度、相关性聚集、财报跳空、流动性、保证金、汇率和税务影响。输出是研究辅助，不是收益保证；不得自动下单，任何动作由用户确认后在 IBKR 自行执行。
