# 新闻证据规则

## 白名单

只使用以下六类英文来源作为新闻事实、事件证据或仓位建议依据：

| 类别 | 角色 | 规范域名 |
|---|---|---|
| SEC EDGAR | 一手监管文件 | `sec.gov` |
| 公司官方 Investor Relations | 一手公司材料 | 发行人控制的官方公司域名 |
| Reuters | 独立专业媒体 | `reuters.com` |
| CNBC | 独立专业媒体 | `cnbc.com` |
| MarketWatch | 独立专业媒体 | `marketwatch.com` |
| Nasdaq | 独立专业媒体 | `nasdaq.com` |

搜索引擎、社交平台、摘要、聚合页和 AI 生成内容只用于发现线索；必须打开最终证据页验证。不得使用新浪、腾讯、东方财富或任何其他中文新闻网站作为新闻来源、事件证据或仓位建议依据。

## 两阶段漏斗与 Token 停止规则

默认时间窗：最近 36 小时；已知未来财报／宏观日历可单列，不混入新闻候选。先做元数据扫描，再做事件簇深读：

1. 候选元数据最多 120 条，只读取标题、主体、时间、来源、URL 和可能命中的持仓／系统因子。
2. 元数据入队顺序固定为：全部实际持仓的直接事件、系统性宏观／政策事件、路径清晰的二阶候选、其他候选。必须先为前两类保留名额；若前两类本身超过 120 条，停止声称完整覆盖并走溢出披露，不得静默截断。
3. 按同一事实与底层报道来源去重；去重后事件簇最多 40 个。优先级队列溢出时同样不得声称完整覆盖。
4. 深读事件簇最多 12 个，顺序为直接、系统性、清晰二阶；其余只保留元数据扫描结论，不打开全文。
5. 每个事件簇最多打开 3 个来源；每个来源最多保留 600 个中文字符或 1,200 个英文字符的事实摘录，不复制全文。
6. 达到双源门槛后停止继续取证：取得“一手 + 独立媒体”或“两家底层来源独立的媒体”即停止打开新来源。
7. 发生数字、时间、指引或事件性质冲突时，冲突升级最多额外打开 2 个来源（限白名单）；总计最多 5 个来源。仍不能裁决则降级，不继续扩张搜索。

溢出披露必须写明：元数据候选数、去重事件簇数、深读数、未深读数、优先级截断原因，以及正式覆盖是否因此未验证。5/4/6 是输出上限，不是输入筛选规则；不得先按输出名额截断候选。

## 四种验证状态

为每个至少存在一个合格白名单来源的候选事件只赋予以下一种状态：

| 状态 | 必要证据 | 可支持仓位动作 |
|---|---|---|
| `verified_primary_plus_independent` | 至少一项 SEC／公司 IR 一手材料，加至少一家独立白名单媒体的一致确认 | 是 |
| `verified_two_independent` | 至少两家相互独立的白名单媒体一致确认，且底层报道来源不同 | 是 |
| `primary_only_pending` | 只有 SEC／公司 IR 一手材料，尚无独立白名单媒体确认；或一手材料与媒体解读存在实质冲突 | 否，仅等待确认 |
| `unverified_single_source` | 只有一家媒体、匿名爆料、传闻，或独立媒体之间存在未解决的实质冲突 | 否，仅观察 |

SEC 与同一公司的 IR 都属于一手证据，不能冒充两家独立媒体。在至少存在一个合格来源候选时，证据不足以达到较高状态才按表中四种状态降级。

若没有任何候选 URL 或来源，走零来源分支：标记 `evidence_gap`，验证状态字段写“不适用”，且不得进入动作依据。`evidence_gap` 是证据缺口，不是第五种验证状态；不得把零来源降成 `unverified_single_source`。

## 规范域名与去重

对每条证据执行以下处理：

1. 将主机名转小写，移除 `www.`、移动版前缀、端口、尾随点、查询参数和追踪参数。
2. SEC 子域归一为 `sec.gov`；媒体子域归一为表中规范域名；公司 IR 保留发行人的官方注册域名并标记 `source_type=issuer_ir`。
3. 记录 `canonical_domain`、原始 URL、发布者、发布时间、抓取时间和底层报道来源。
4. 按“同一事件事实 + 底层原始报道来源”去重。转载、联合供稿、同一通讯社在不同站点的镜像只计一份证据。
5. Nasdaq 或其他白名单站点若转载 Reuters 内容，证据仍只计 Reuters 一源；不得因两个 URL 提升验证状态。

## 冲突处理

- 保留每个冲突陈述、链接、发布时间和差异点，不静默选择有利叙述。
- 一手材料与媒体解读冲突时，以一手材料界定已确认事实，但将争议结论降为 `primary_only_pending`。
- 两家独立媒体实质冲突且无一手材料裁决时，降为 `unverified_single_source`。
- 时间较新的正式更正可取代旧说法；同时保留更正轨迹，并解释状态变化。
- 数字、时间、指引范围或事件性质不能对齐时，不得将事件写入动作依据。

## 每条事件证据字段

至少记录：事件标题、事件时间、验证状态、来源名称、`canonical_domain`、原始链接、发布时间、抓取时间、支持的事实、冲突／更正说明。把事实、媒体解读和分析推论分开书写。
