# Galaxy Buffett：IBKR 插件唯一持仓输入设计

日期：2026-07-21
状态：待用户书面确认

## 目标

将 `Galaxy Buffett - Daily Stock Analysis` 的正式账户数据源改为 Interactive Brokers (IBKR) 插件。每日晨报不再读取 Flex CSV、Elasticsearch、本地缓存、历史对话或用户口述来认定正式持仓。

该变更只重构持仓输入与覆盖判断。英文新闻白名单、交叉验证门槛、HTML 八章节、最多 5/4/6 项输出限制和禁止自动交易的规则保持不变。

## 数据源边界

正式账户事实只接受当前会话中已授权的 `Interactive Brokers (IBKR)` 插件返回值。技能按工具能力选择接口，不依赖内部哈希工具名。

每天固定执行三次只读调用：

1. **全部未平仓持仓**：数量、市场价、市值、平均成本、当日盈亏、未实现盈亏、币种、资产类型和合约标识。
2. **账户财务指标**：净清算值、现金、可用资金、购买力、初始保证金、维持保证金、总持仓市值和杠杆。
3. **分币种余额**：现金、已结算现金、股票市值、净清算值、汇率和盈亏。

禁止使用以下内容填补插件缺值：

- `read_ibkr_snapshot.py`；
- Elasticsearch 索引；
- Flex CSV 或后台配置文件；
- 上一份晨报或历史对话中的持仓；
- 用户口述证券范围。

用户口述的证券仍可作为临时观察范围，但必须与插件正式持仓分开显示。

## 调用流程与 Token 控制

1. 先调用持仓接口一次。
2. 持仓调用成功后，并行或连续各调用一次账户指标和分币种余额。
3. 三项调用在同一报告运行中复用结果，不为每条新闻或每个证券重复读取账户。
4. 只有新闻初筛命中重点持仓且现有持仓行情不足以回答影响时，才调用额外市场数据；最多覆盖最终展开的 6 个重点持仓。
5. 不读取成交、活动订单、观察列表或历史收益，除非用户在当次任务明确要求。
6. 只对明确的临时网络或服务错误重试一次；认证、权限或账户选择错误直接进入降级分支。

插件中的创建交易指令、提交订单、修改观察列表和其他写操作不属于该技能流程，任何情况下均不得调用。

## 规范化快照

三项插件结果在内存中合并为一次报告级快照，不保存授权凭证或账户标识：

| 规范字段 | 插件字段或规则 |
|---|---|
| `symbol` | `contract_description`；缺失或歧义时保留“未提供”，必要时条件调用合约搜索核实 |
| `contract_id` | `contract_id`，仅用于同次运行内精确识别证券 |
| `asset_class` | `asset_class` |
| `quantity` | `position` |
| `mark_price` | `market_price` |
| `market_value` | `market_value` |
| `average_cost` | `average_price` |
| `cost_basis` | 插件未直接返回时写 `null`，不以数量乘均价推算 |
| `unrealized_pnl` | `unrealized_pnl` |
| `daily_pnl` | `daily_pnl` |
| `currency` | `currency` |
| `portfolio_weight` | `market_value / net_liquidation`，两者均为有效数值时才计算 |
| `fetched_at` | 本次插件工具调用完成时间 |

持仓只保留数量或市值非零的项目，并按绝对市值降序、证券代码升序排列。正式覆盖数为插件返回的非零持仓数。

不得输出账户号码、授权信息、内部会话、原始工具响应或可用于重新授权的标识。

## 状态与失败处理

### `ready`

持仓、账户指标和分币种余额三项均成功，且在同一次报告运行中取得。允许完整计算组合权重、现金比例、集中度和保证金风险。

### `partial`

持仓成功，但账户指标或分币种余额至少一项失败。实际持仓可以正式扫描；缺失字段显示“未验证”。允许输出持仓相关事实和“观察／持有”，禁止仓位百分比、集中度再平衡和条件式加减仓。

### `empty`

持仓接口成功并明确返回空数组。显示“账户当前没有未平仓持仓”，不把它当成数据故障。

### `source_unavailable`

持仓接口因未授权、权限、服务或网络问题失败。正式持仓覆盖显示“未验证”，停止所有持仓建议，不尝试本地数据回退。

### 新鲜度

记录插件查询完成时间，不将它伪装成券商估值时间。三项固定调用成功且晨报在最后一次调用后 15 分钟内生成时标记 `fresh`；超过 15 分钟标记 `stale`。插件未返回有效结果或无法记录查询时间时标记 `unknown`。

## 晨报呈现

“数据与覆盖”章节必须显示：

- 数据源：`Interactive Brokers (IBKR) plugin`；
- 插件查询时间；
- `ready / partial / empty / source_unavailable`；
- `fresh / stale / unknown`；
- 已扫描持仓数／正式持仓总数；
- 账户指标或币种余额的缺失情况；
- 报告生成时间。

正式持仓建议卡保留数量、市值、权重、影响类别、建议、触发条件、反证条件、时间范围、风险、证据状态、链接和理由。插件没有返回的字段填写“未提供／未验证”，不得推算。

## 测试设计

修改前先运行当前技能基线，证明它会要求本地 Elasticsearch 并忽略已授权插件。

修改后验证以下场景：

1. 插件三项调用成功：状态为 `ready`，正确计算权重并扫描全部非零持仓。
2. 持仓成功、账户指标失败：状态为 `partial`，仍扫描持仓但不生成仓位动作。
3. 持仓返回空数组：状态为 `empty`，不回退本地文件。
4. 持仓调用失败：状态为 `source_unavailable`，不调用 Elasticsearch、CSV 或旧脚本。
5. 插件缺失成本、价格或币种：保留空值，不推算。
6. 同一晨报只执行三次固定账户调用，不按证券重复调用。
7. 新闻未命中重点持仓：不额外调用行情工具。
8. 任何场景都不调用创建指令、提交订单或其他写工具。
9. HTML 仍保留八个章节、英文证据链接和完整风险披露。

## 修改范围

计划修改并同步安装：

- `SKILL.md`：主流程改成 IBKR 插件唯一输入；
- `references/ibkr-input-contract.md`：重写为插件调用、字段映射和状态合同；
- `references/portfolio-analysis.md`：适配 `ready/partial` 与插件字段；
- `references/morning-report-contract.md`：用插件查询时间替代文档批次时间；
- 测试与前向场景：覆盖插件成功、部分失败、空仓和不可用。

现有 `scripts/read_ibkr_snapshot.py` 暂时保留为未引用的历史文件，不再由技能调用。本次不删除文件，避免未经确认的破坏性操作。

项目目录与 `~/.codex/skills/galaxy-buffett-daily-stock-analysis` 的已安装副本必须保持逐文件一致。

## 验收标准

- 真实运行只通过 IBKR 插件取得正式持仓；
- 插件不可用时明确失败且不回退；
- 平衡模式固定三次只读账户调用；
- 账户指标完整时能够计算真实组合权重与现金风险；
- 没有自动交易、凭证输出或隐式写操作；
- Token 上限、新闻证据规则和 HTML 合同保持有效；
- 项目版、安装版和测试全部通过验证。
