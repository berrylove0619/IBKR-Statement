# Galaxy Buffett 持仓大白话总结设计

## 目标

解决个股持仓卡片信息完整但结论过深的问题。用户应先在 20 秒内看懂每只持仓“现在怎么做、为什么、什么时候可以买、什么时候应该卖”，再按需展开八角色详细分析。

## 页面结构

每个 `article.holding-card` 在持仓事实之后、所有事件与角色分析之前，固定显示一个醒目的 `section.plain-language-summary`：

1. `h4` 标题固定为“持仓建议·大白话总结”。
2. 显示三个动作字段：`当前动作`、`买入建议`、`卖出建议`。
3. 显示唯一一个 `p.plain-summary-text`，可见中文正文为 100–200 个字符。
4. 总结必须说明当前结论、核心原因、主要仓位风险，以及下一项可验证条件；使用普通投资者可直接理解的语言，不堆砌术语和证据编号。
5. 既有四字段快速摘要继续显示；八角色详细分析放入 `details.committee-details`，默认折叠，标题为“展开八角色详细分析与证据”。
6. 简版持仓也必须提供同样的大白话总结，但不需要伪造八角色分析。

## 动作表达

`当前动作`继续使用 Skill 已有动作枚举。`买入建议`和`卖出建议`必须明确回答是否适合行动，并同时遵守账户状态与证据门槛：

- `ready + fresh` 且事件证据达标：可给出条件式加仓或减仓，必须写明可验证触发条件。
- `partial / stale / unknown`：买入建议只能是“暂不加仓／等待确认”，卖出建议只能是“暂不主动卖出／仅观察风险”，不得输出条件式交易规模或价位。
- `source_unavailable / empty`：不生成个人买卖建议。

“买入建议”和“卖出建议”是研究结论，不是订单。不得使用“立即买入”“全部卖出”等无条件指令，也不得自动调用交易工具。

## 当前报告迁移

为 V3 报告的 MU、GOOG、NVDA、DRAM、NOK、SPCX、IBKR 七张卡片分别编写 100–200 字总结。当前账户为 `partial`，因此七张卡片均不得给出条件式加仓或减仓；重点说明等待财报、数据对账、集中度或事件验证。

## Skill 与公开版本

同步更新：

- 本地 Skill 的持仓分析规则与晨报输出合同；
- Skill 主流程中的持仓卡片要求；
- GitHub 公开包中的同名 Skill 文件和方法说明；
- 自动验证器与测试夹具。

个人持仓 HTML 只保存在本地，不加入公开 GitHub 包。

## 验证标准

自动校验每张 `holding-card`：

- 恰好一个 `plain-language-summary`；
- 恰好一个 `plain-summary-text`；
- 可见文本长度为 100–200 个字符；
- 包含“当前动作”“买入建议”“卖出建议”；
- 深度卡包含默认关闭的 `details.committee-details`；
- 原有新闻数量、持仓覆盖、证据字段和隐私校验继续通过。

人工检查桌面与移动阅读顺序：持仓事实 → 大白话结论 → 四字段快速摘要 → 可选详细分析。
