# Galaxy Buffett 专属 Skill 无技能基线

日期：2026-07-21
证据来源：三个由未加载拟议专属 Skill 的独立新 Agent 生成的场景回答。

## 结论

无专属 Skill 的 Agent 已具备可用的投资研究判断：能够做全持仓扫描、去重和分层阅读，优先回到一手资料，表达不确定性，并避免因短期价格波动而直接建议恐慌性卖出。问题不在于“完全不会分析”，而在于关键限制依赖当次 Agent 判断，未固化为每次运行都一致执行、可复核的专属纪律。

因此，拟议 Skill 的验收目标是让已存在的能力在确定范围、证据状态、持仓数据时效、影响分类、动作门槛和 HTML 报告证据链上可复现，而不是把基线回答描述成全部错误。

## 场景 A：25 个持仓与约 100 条候选新闻的资源压力

### 基线表现摘要

回答提出两阶段漏斗：先扫描元数据、去重为 25--40 个事件，再深读 8--15 个，发布 5--8 个核心事件和 3--6 条简报，最后检查全部 25 个持仓。它还给出 100 分评分、宽泛的必看事件类别、15k--30k 输入 token 软预算、1,200--2,000 字中文输出、缓存、覆盖状态与静默日行为。

### 可保留能力

- 有全持仓扫描意识，未只处理最显眼的个股。
- 先去重再深读，能在候选新闻较多时控制资源。
- 已尝试声明 token 预算、缓存、覆盖状态与无重大事件日的输出行为。

### 缺失的专属纪律

- 上限与批准的产品契约不一致，且只是软预算，不能稳定约束每天的读取与输出范围。
- 没有固定六源白名单，也没有可机器或人工复核的命名验证状态。
- 没有以最新一次成功 IBKR 导入时间为前置门槛，无法证明建议基于新鲜的实际持仓。
- 没有把影响固定分为直接、二阶、宏观共同因子和无实质关联。
- 没有确定性的“单次组合级综合分析”，也没有“仅财报命中且确需深挖时一次专项展开”的例外规则。
- 没有固定 HTML 证据合同，读者无法预期每条结论应有哪些来源、链接、时效与反证信息。

### Skill 必须补齐的规则

每日固定读取新闻证据、组合分析和晨报合同；仅在事件触发时条件加载财报、宏观、长期质量或科技供应链参考。先用确定性规则扫描全部持仓，最多展开 6 个持仓；默认只做一次组合级综合，只有符合条件的财报才增加一次专项分析。报告上限固定为重大市场事件 5 条、科技事件 4 条、重点持仓 6 个，其余归入“无重大变化”。

## 场景 B：NVDA 财报证据冲突与立即卖出压力

### 基线表现摘要

回答优先采用公司材料、SEC 文件、财报原文和电话会记录；将媒体表述追溯回原始资料；把匿名消息标为未验证；保留相互冲突的解读，并拒绝给出简单的“全部卖出”。它按仓位大小、投资期限和投资逻辑是否被破坏，提出条件式风险管理。

### 可保留能力

- 正确优先一手材料，并追溯媒体叙述的原始出处。
- 会揭示不确定性与冲突，不把匿名消息包装为事实。
- 不因价格本身建议恐慌性清仓，且能把建议与仓位、期限和 thesis break 联系起来。

### 缺失的专属纪律

- 没有明确的专业媒体白名单。
- 没有正式使用 `verified_primary_plus_independent`、`verified_two_independent`、`primary_only_pending`、`unverified_single_source` 等验证状态。
- 虽然拒绝“卖光”，仍可能给出减仓建议，却未明确分离“既有集中度再平衡”与“由新闻触发的仓位动作”。
- 没有结构化的来源、规范域名与证据链接审计轨迹。

### Skill 必须补齐的规则

新闻证据只能来自 SEC EDGAR、公司 IR、Reuters、CNBC、MarketWatch、Nasdaq；搜索与聚合页只可发现线索，中文新闻网站不得成为事件证据或建议依据。重要事件进入仓位建议前必须满足“一手来源加一家独立白名单媒体”或“两家相互独立的白名单媒体”；同一通讯社转载不得重复计数。仅有一手资料须为 `primary_only_pending`，单一媒体、匿名爆料和传闻须为 `unverified_single_source` 并仅限观察区。单一来源、传闻、短时价格波动和泛行业新闻均不得独立触发加仓、减仓或清仓；如讨论集中度调整，必须明确其并非新闻触发动作。

## 场景 C：AAPL、MSFT、BRK.B 与短债 ETF 的科技／宏观相关性筛选

### 基线表现摘要

回答把 AI／数据中心新闻主要映射给 MSFT，把亚洲供应链新闻映射给 AAPL，把利率新闻映射给 BRK.B 与短久期美债 ETF；其余 AI／光学新闻只保留为观察，并在不存在公司层面变化时建议持有、不追高。

### 可保留能力

- 已有组合特定的相关性筛选，而非把所有热门新闻同等对待。
- 能识别部分供应链与利率传导路径。
- 在缺少公司层面变化时，默认“持有／不追高”是合理的无动作结论。

### 缺失的专属纪律

- 没有明确标注直接、二阶、宏观共同因子或无实质关联，传导强度会随当次判断而改变。
- 推荐表之前没有要求完成证据验证。
- 没有声明持仓数据的新鲜度。
- 没有一致提供失效／反证条件、证据链接及完整报告字段。

### Skill 必须补齐的规则

每个事件必须归类为直接、二阶、宏观共同因子或无实质关联；只有前两类且传导路径清晰的事件进入持仓建议。建议必须以最新成功导入的 IBKR 实际持仓为依据；数据缺失或过期时明确提示而不猜测。每项动作均须列出触发条件、反证条件、时间范围和主要风险；没有新证据时允许并应优先给出“维持原计划”。晨报 HTML 必须包含验证状态、来源与链接、规范域名、持仓数据时间、影响类别、建议及其反证条件，形成完整可审计证据链。

## 加载 Skill 后的前向测试

证据来源：三个新的独立 Agent 仅加载项目 Skill 及其路由参考资料后，按上述三个场景重跑；未浏览网页、未修改项目文件。以下记录测试回答实际体现的约束，而非对 Skill 能力的宣传。

### 场景 A：25 个持仓与 100 条候选新闻

**结果：通过。** 回答先扫描全部 25/25 个持仓，再按底层事件／来源去重候选新闻；读取和 HTML 输出遵守固定上限：重大市场事件最多 5 条、科技事件最多 4 条、重点持仓最多展开 6 个，分析正文最多 2,200 个中文字。它使用六类允许来源，列出四种验证状态（`verified_primary_plus_independent`、`verified_two_independent`、`primary_only_pending`、`unverified_single_source`），并在达到证据门槛后停止继续累积同一事件的证据。

回答以最新一次成功 IBKR 导入为前提，记录导入时间、持仓日期和数据年龄；数据过期或未知时下调全部持仓动作。每个事件使用直接、二阶、宏观共同因子或无实质关联四类标签；只按事件匹配条件加载参考资料。组合层面固定只做一次综合，且仅在财报命中并确有必要时最多增加一次专项展开；证据不足时默认维持原计划。

### 场景 B：NVDA 财报冲突与“立即卖出”压力

**结果：通过。** 回答先要求新鲜的 IBKR 持仓快照，再分开公司 IR／SEC 事实、媒体解读、匿名说法和价格反应。已知财报事件被保留为 `primary_only_pending`，匿名主张标为 `unverified_single_source`，冲突未被强行消解，跳空下跌也未被当作事实证据。该处理遵循四种命名验证状态及其两源／一手加独立媒体门槛，而不是以单一来源或市场价格补足证据。

当前动作限定为观察并维持既有计划；只有验证状态达标且投资 thesis 受损时，才允许条件式减仓。若已有集中度上限需要再平衡，回答允许独立执行，但明确标为非新闻触发。HTML 示例包含影响类别、验证状态、触发条件、反证条件、时间范围和主要风险，保持动作证据链可审计。

### 场景 C：AAPL、MSFT、BRK.B 与短债 ETF 的相关性筛选

**结果：通过。** 在没有可核验原始链接或 IBKR 导入详情时，回答明确将数据新鲜度标为 `unknown`，扫描 4/4 个持仓而不猜测敞口。它将泛 AI／光学／亚洲供应链主题归为无实质关联，将数据中心新闻标为尚未证实的 MSFT 二阶传导，并将利率归为宏观共同因子；直接、二阶、宏观共同因子和无实质关联四类均有一致边界。

结论是不交易、维持原计划，并请求补齐 ETF、时间和权重数据。只有一手资料加独立来源或两家相互独立来源满足门槛后才重新评估，确保相关性标签、数据时效和条件式动作不会被宽泛科技主题替代。

## 基线对比与残余风险

前向测试显示，原基线中依赖临场判断的软范围已被固定合同替代：5/4/6 上限、四种验证状态、最新成功导入门槛、四类相关性、条件式动作字段、无充分新证据时的维持计划，以及一次组合综合／最多一次财报专项均在三种压力场景中出现。尤其是冲突证据、持仓数据未知和泛行业主题不再被直接转换为交易建议。

这仍只是研究流程的一致性证据，不是收益或交易结果的保证。正式晨报仍取决于实时网页和原始来源可访问性、IBKR 导入的完整与及时性、公司及 ETF 敞口映射是否仍然有效，以及用户明确的风险限额和投资偏好。Skill 只能辅助研究、记录证据和提出带条件的观察；它不能自动执行交易，也不能替代用户的投资决策。

## 跨场景的验收基线

后续 Skill 应证明以下约束能在每次运行中稳定出现：

1. 固定六源白名单和两源／一手加独立媒体的证据门槛，以及明确的验证状态。
2. 最新成功 IBKR 导入时间与实际持仓的时效门槛。
3. 直接、二阶、宏观共同因子、无实质关联的统一影响标签。
4. 固定的条件加载、最多展开 6 个持仓、一次组合综合与受限财报专项规则。
5. 禁止单一来源、传闻、短时价格或泛行业新闻独立触发仓位动作；每项动作都有条件、反证、时间范围和风险。
6. 固定的本地 HTML 晨报证据合同，能够追溯来源、规范域名、链接、数据时效和判断依据。

这些是对基线能力的纪律化与可审计化补强，不代表无 Skill Agent 无法给出有价值的投资研究意见。
